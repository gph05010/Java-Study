# 🍔 Java 햄버거 키오스크 (ArrayList vs HashMap)

## 📅 날짜
2026.01.30

## 📝 학습 주제
- ArrayList를 활용한 데이터 관리
- 중복 데이터 처리 로직 구현 (직접 구현 vs HashMap)

## 🚀 트러블 슈팅 (오늘의 삽질)
### 상황
주문 목록에서 동일한 메뉴가 들어왔을 때, 리스트에 계속 추가되는 것이 아니라 "치즈버거 2개"처럼 수량만 카운트하고 싶었음.

### 문제
처음에는 `HashMap`을 배우기 전이라 `ArrayList`만을 사용하여 구현을 시도함.
1. 중복되지 않은 메뉴를 담을 임시 리스트(`orderList2`) 생성
2. 원본 리스트를 순회하며 중복 여부 체크 (`isFind` 플래그 사용)
3. 중복이 없으면 추가, 이후 다시 카운팅 루프 실행

이 과정에서 2중 for문을 사용하며 인덱스 에러(`IndexOutOfBoundsException`)와 출력 형식 에러(`UnknownFormatConversionException`) 등을 겪음.

### 해결 및 깨달음
- 직접 2중 반복문으로 구현하며 논리적 사고력을 기름.
- 이후 강사님을 통해 `HashMap`의 `getOrDefault` 메서드를 배우고, 10줄이 넘는 코드가 단 1줄로 줄어드는 것을 경험함.
- 결론: 자료구조(HashMap)를 왜 써야 하는지 뼈저리게 느낌.
